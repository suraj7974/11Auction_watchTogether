     Channel: one per room, room:{code}.

     1. Playback sync (host-authoritative):
       - Host's YouTube player is the truth. On host play / pause / seek
     (detected via IFrame
     onStateChange + position-jump detection), host (a) writes {is_playing,
     position_seconds, playback_updated_at} to rooms and (b) broadcasts playback
     event.
       - Clients apply the event: seek to position_seconds + (now − emittedAt if
     playing), then
     play/pause. Non-host players are kept in a "follow" mode.
     2. Drift correction: host broadcasts a lightweight sync heartbeat every ~5s
     with current
     position; any client whose drift > ~1.5s re-seeks. This is what guarantees
     "multiple users
     should not permanently desync."
     3. Late-joiner sync: on join, read rooms playback fields → compute current
     position from
     playback_updated_at; also broadcast request_sync → host replies with a
     fresh sync.
     Baseline from DB, refined by host reply.
     4. Chat: insert into messages; render live via postgres_changes
     subscription
     (persisted + realtime). Survives refresh and seeds room history.
     5. Participants: Supabase presence (track {user_id, name, color, role}) →
     live list
     with join/leave. room_participants table persists membership/role.
     6. Reactions: broadcast reaction (emoji) — ephemeral, high-frequency, not
     persisted.
     7. Countdown: host broadcasts countdown (3→2→1) → all clients show overlay,
     then host
     triggers synchronized play.
